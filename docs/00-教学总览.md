# DevFlow 教学总览

## 教学定位

本系列文档面向 **Agent 实习开发面试**，以 DevFlow（原 MewCode）的约 20,000 行 Python 源码为教材，逐模块讲解 Agent Harness 的架构设计、实现细节和面试应答技巧。

## 教学原则（继承自 OpsPilot 教学）

每讲一个模块按 3 层递进：

1. **怎么用**（5%）——快速过入口和调用方式
2. **怎么实现**（35%）——逐文件精读关键代码路径，讲数据流
3. **为什么这样设计**（60%）——面试的核心

对于"为什么"，建立固定分析框架：

```
设计方案 A（DevFlow 选的） vs 方案 B（替代方案）
├── 在什么约束下做决策？
├── A 比 B 好在哪？
├── A 的代价是什么？（没有免费午餐）
└── 如果面试官追问"为什么不用 B"，你怎么回答
```

每个模块结尾附 **面试话术模板**——可以直接在面试中使用的叙述框架。

## 推荐学习顺序

### 第一轮：核心链路（必须掌握）

```
01 Agent 核心循环 → 02 上下文窗口管理 → 03 工具系统设计 → 04 权限与安全模型
```

这四个模块覆盖了 Agent Harness 最核心的"输入→推理→工具→输出"全链路。面试时如果只讲一个，讲 Agent 核心循环。

### 第二轮：扩展能力（丰富面试素材）

```
05 多 Agent 协作 → 06 MCP 协议集成 → 07 记忆系统
```

这三个模块证明你理解 Agent 的高级能力——多 Agent、外部工具集成、持久化记忆。

### 第三轮：框架开放性（展示架构思维）

```
08 Hooks 扩展机制 → 09 命令系统
```

这两个模块展示你对"框架如何让用户扩展"的理解——开放性 vs 可维护性的平衡。
